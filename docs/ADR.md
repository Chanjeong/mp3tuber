# Architecture Decision Records

## 철학
로컬·개인용 도구로서 **작동하는 최소 구현(MVP)을 빠르게**. 외부 의존성과 운영 복잡도를 최소화하고, 비가역적·법적 리스크가 있는 선택(공개 배포 등)은 피한다. 보안(키 비노출·커맨드 인젝션 차단)과 일관성(에러 모델·디자인 토큰)은 타협하지 않는다.

---

### ADR-001: Next.js App Router + Node 런타임 route handler
**결정**: Next.js 15 App Router를 쓰고, `/api/search`·`/api/convert`는 `runtime = 'nodejs'` route handler로 구현한다.
**이유**: 검색은 키를 서버에 숨겨 호출해야 하고, 변환은 `child_process`(yt-dlp/ffmpeg)와 `fs`(temp 파일)·스트림이 필요하다 — 모두 Node 런타임에서만 가능. App Router의 Server Component로 키·로직을 서버에 격리.
**트레이드오프**: Edge 런타임의 저지연·글로벌 배포 이점을 포기. 로컬 전용이라 무의미하므로 수용.

### ADR-002: 변환 엔진 = youtube-dl-exec(yt-dlp) + ffmpeg-static
**결정**: 변환은 `youtube-dl-exec`(yt-dlp 바이너리 래퍼) + `ffmpeg-static`(ffmpeg 바이너리 번들)로 한다.
**이유**: yt-dlp는 YouTube 변경에 가장 잘 대응하는 사실상 표준 도구. `ffmpeg-static`이 플랫폼별 바이너리를 제공해 **수동 설치 없이** mp3 추출/mp4 merge 가능. 순수 JS 라이브러리(ytdl-core 류)보다 안정적.
**트레이드오프**: 설치 시 yt-dlp 바이너리를 GitHub에서 받는 의존(사내 프록시/방화벽 환경에서 실패 가능 → 수동 설치 fallback). 외부 바이너리 실행에 따른 리소스 소비.

### ADR-003: 로컬 전용 — 공개 배포하지 않음
**결정**: 앱은 로컬(`npm run dev`)에서 개인용으로만 실행하고 공개 배포하지 않는다.
**이유**: YouTube 스트림 리핑은 ToS 위반이며, 공개 서비스화 시 API 키/계정 정지·법적 리스크가 크다. 개인·학습 목적에 한정해 리스크를 회피.
**트레이드오프**: 타인 사용·원격 접근 불가. 의도된 제약.

### ADR-004: 연관 검색어(자동완성) 제외
**결정**: 검색 자동완성/연관 검색어 기능을 넣지 않는다.
**이유**: 공식 Data API에 자동완성이 없어 비공식 `suggestqueries` 엔드포인트에 의존해야 하고(불안정·비공식), 매 키 입력마다 검색을 호출하면 **쿼터(검색당 100유닛, 일 100회)**가 폭증한다.
**트레이드오프**: 입력 편의 저하. 제출 기반 검색으로 단순화하며 수용.

### ADR-005: 변환 결과는 브라우저 다운로드 (라이브러리 미도입)
**결정**: 변환물은 `Content-Disposition: attachment`로 브라우저가 바로 내려받게 하고, 앱 내 보관함/히스토리는 만들지 않는다.
**이유**: MVP 단순화. 파일 관리는 OS/브라우저에 위임.
**트레이드오프**: 재다운로드·목록·재생 불가(매번 재변환). 후속 확장 여지로 남김.

### ADR-006: Tailwind v4 @theme + tokens.css 단일 소스
**결정**: 디자인 값은 `src/styles/tokens.css`(CSS custom properties)에 정의하고 Tailwind v4 `@theme`로 semantic 유틸에 매핑한다.
**이유**: 토큰 단일 소스로 디자인 표류(drift)를 원천 차단(`docs/UI_GUIDE.md`). 한 곳만 바꾸면 전체 일관 변경.
**트레이드오프**: 초기 토큰/유틸 정의 비용. 일관성 이득이 크므로 수용.

### ADR-007: 통일 에러 모델 + 에러 코드 카탈로그
**결정**: 모든 JSON 에러를 `{ error: { code, message } }`로 통일하고, 코드 카탈로그를 `docs/API.md`에 정의한다. 클라엔 안전한 메시지+code만, 상세·키·스택은 서버 로그로만.
**이유**: 클라이언트 분기·UX 메시지·디버깅을 일관되게. 민감정보 노출 방지.
**트레이드오프**: 코드 카탈로그 유지 비용. 일관성·보안 이득으로 정당화.

### ADR-008: 변환은 요청-응답 내 동기 처리 (잡 큐 없음)
**결정**: 변환을 별도 잡 큐/워커 없이 `/api/convert` 요청 처리 중 동기로 끝내고 스트리밍 응답한다.
**이유**: 단일 사용자·MVP에 큐는 과설계. 요청 1건 = 변환 1건으로 단순.
**트레이드오프**: 진행률 표시·재시도 큐 없음, 장시간/라이브 영상은 타임아웃으로 방어. 동시 변환 다수 시 리소스 부담 → 비권장으로 안내.
