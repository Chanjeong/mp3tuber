# PRD: mp3tuber

## 목표
YouTube 영상을 키워드로 검색해 원하는 영상을 고르고, 이를 **mp3(오디오) 또는 mp4(비디오)로 변환해 로컬에 내려받는** 개인용 도구. "검색 → 선택 → 변환 → 저장"을 한 화면에서 끝내는 것이 핵심.

## 사용자
- 본인(개인 사용자). 로컬(`npm run dev`)에서 단독으로 사용.
- 기술 배경: 프론트엔드 개발자. 빠르고 군더더기 없는 도구형 UX를 선호.

## 핵심 기능
1. **키워드 검색** — 검색창에 입력 후 제출하면 관련 YouTube 영상 목록을 가져온다.
2. **결과 목록** — 각 결과를 카드로 표시: **썸네일 · 제목 · 채널명**. (영상 길이는 MVP 제외 — 아래 "알려진 한계" 참조)
3. **변환** — 카드에서 포맷(`mp3`/`mp4`)을 선택하고 변환 버튼을 누르면 서버가 yt-dlp+ffmpeg로 변환한다.
4. **다운로드** — 변환이 끝나면 브라우저가 파일을 내려받는다(`Content-Disposition: attachment`). 파일명은 영상 제목 기반(sanitize).

## 사용자 시나리오 (정상 + 실패 경로)
- **정상**: 검색어 입력 → 결과 카드 표시 → mp3 선택 → 변환 버튼 → 잠시 로딩 → 파일 다운로드 → 재생 확인.
- **결과 없음**: 매칭 영상이 없으면 "검색 결과 없음" 안내(에러 아님).
- **검색 오류**: 네트워크/쿼터/키 오류 시 에러 배너 + 재시도 버튼.
- **변환 로딩**: 변환 중에는 해당 카드 버튼이 비활성 + 스피너, 다른 카드는 정상 사용.
- **변환 실패**: 연령제한·비공개·지역차단·DRM 등으로 실패하면 카드에 인라인 에러 + 재시도.

## 비기능 요구
- 변환 소요는 영상 길이에 따라 **수 초 ~ 수십 초** 허용(동기 처리, 진행률 표시는 MVP 제외).
- 단일 사용자 가정. 동시 변환은 yt-dlp/ffmpeg 프로세스·CPU·디스크를 소모하므로 **과도한 동시 변환은 비권장**.
- 큰 파일도 메모리에 적재하지 않고 **스트리밍**으로 응답.
- **YouTube Data API 쿼터**: `search.list`는 호출당 100 유닛, 기본 일일 쿼터 10,000 유닛 ≈ **검색 약 100회/일**. 불필요한 중복 검색을 자제한다.

## MVP 제외 사항
- 연관 검색어/자동완성 (비공식 API 의존 회피 + 검색 쿼터 폭증 방지)
- 검색 페이지네이션 / "더 보기" (`nextPageToken` 미사용)
- 변환 라이브러리·히스토리·재생 (다운로드 후 파일은 브라우저가 관리)
- 변환 진행률(SSE/WebSocket) 표시
- 인증/계정, 플레이리스트
- 공개 배포 (ToS·법적 리스크)
- 연령제한 우회(쿠키 주입 등)

## 알려진 한계
- **영상 길이 미표시**: `search.list`는 길이를 반환하지 않고, 길이를 얻으려면 `videos.list` 별도 호출(쿼터·복잡도 증가)이 필요해 MVP에서 제외.
- 연령제한·비공개·지역차단·DRM 영상은 변환에 실패할 수 있다.
- yt-dlp는 YouTube의 내부 변경에 따라 깨질 수 있다 → 필요 시 `youtube-dl-exec`/yt-dlp 업데이트.

## ToS 디스클레이머
개인·학습용으로만 사용한다. 업로더의 저작권을 존중하며, 변환물의 재배포·공개 서비스화는 하지 않는다.

## 디자인
- 다크 테마 고정, 무채색 + 상태색(보라/인디고 금지). 도구형 대시보드 지향(마케팅 페이지 아님).
- 모든 색·타이포·간격·radius는 semantic token으로만(`docs/UI_GUIDE.md`). 좌측 정렬 기본, 콘텐츠 max-width 토큰 사용.
